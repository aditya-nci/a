Database Schema & Data

Table Structure

The database consists of the following key tables:

  • customers: Customer demographics and registration data
  • orders: Order information including timestamps and customer IDs
  • menu_items: Menu items with prices, categories, and descriptions
  • order_items: Junction table linking orders to menu items
  • payments: Payment information and transaction details
-- Sample table structure CREATE TABLE customers ( customer_id INT PRIMARY KEY, first_name VARCHAR(50), last_name VARCHAR(50), email VARCHAR(100), registration_date DATE ); CREATE TABLE orders ( order_id INT PRIMARY KEY, customer_id INT, order_date DATETIME, total_amount DECIMAL(10,2), FOREIGN KEY (customer_id) REFERENCES customers(customer_id) );

Key Business Questions Analyzed

1. Customer Ordering Behavior

Question: What are the ordering patterns of our customers?

-- Average order frequency and value by customer WITH customer_metrics AS ( SELECT c.customer_id, c.first_name, c.last_name, COUNT(o.order_id) AS total_orders, AVG(o.total_amount) AS avg_order_value, SUM(o.total_amount) AS total_spent, DATEDIFF(MAX(o.order_date), MIN(o.order_date)) AS customer_lifespan FROM customers c JOIN orders o ON c.customer_id = o.customer_id GROUP BY c.customer_id, c.first_name, c.last_name ) SELECT *, CASE WHEN total_orders >= 10 THEN 'High Frequency' WHEN total_orders >= 5 THEN 'Medium Frequency' ELSE 'Low Frequency' END AS customer_segment FROM customer_metrics ORDER BY total_spent DESC;

2. Menu Performance Analysis

Question: Which menu items are most popular and profitable?

-- Menu item performance with ranking SELECT m.item_name, m.category, m.price, COUNT(oi.order_item_id) AS times_ordered, SUM(oi.quantity) AS total_quantity, SUM(oi.quantity * m.price) AS total_revenue, RANK() OVER(ORDER BY COUNT(oi.order_item_id) DESC) AS popularity_rank, RANK() OVER(ORDER BY SUM(oi.quantity * m.price) DESC) AS revenue_rank FROM menu_items m JOIN order_items oi ON m.item_id = oi.item_id GROUP BY m.item_id, m.item_name, m.category, m.price ORDER BY times_ordered DESC;

3. Revenue Trends Analysis

Question: How has revenue changed over time?

-- Monthly revenue trends with growth rates WITH monthly_revenue AS ( SELECT DATE_FORMAT(order_date, '%Y-%m') AS month, SUM(total_amount) AS revenue, COUNT(order_id) AS order_count, AVG(total_amount) AS avg_order_value FROM orders GROUP BY DATE_FORMAT(order_date, '%Y-%m') ) SELECT month, revenue, order_count, avg_order_value, LAG(revenue) OVER(ORDER BY month) AS prev_month_revenue, ROUND( (revenue - LAG(revenue) OVER(ORDER BY month)) / LAG(revenue) OVER(ORDER BY month) * 100, 2 ) AS growth_rate_percent FROM monthly_revenue ORDER BY month;

Advanced SQL Techniques Used

Window Functions

Extensive use of window functions for advanced analytics:

  • ROW_NUMBER(): Customer ranking by total spending
  • RANK() and DENSE_RANK(): Menu item performance ranking
  • LAG() and LEAD(): Time-series analysis for growth rates
  • NTILE(): Customer segmentation into quartiles

Common Table Expressions (CTEs)

Complex queries broken down using CTEs for better readability and maintenance:

-- Multi-level CTE for complex analysis WITH customer_orders AS ( -- First level: Basic customer metrics SELECT customer_id, COUNT(*) AS order_count FROM orders GROUP BY customer_id ), customer_segments AS ( -- Second level: Segmentation SELECT customer_id, order_count, NTILE(4) OVER(ORDER BY order_count) AS segment FROM customer_orders ) SELECT segment, COUNT(*) AS customer_count, AVG(order_count) AS avg_orders FROM customer_segments GROUP BY segment;

Stored Procedures

Created reusable stored procedures for common business queries:

-- Stored procedure for customer analysis DELIMITER // CREATE PROCEDURE GetCustomerAnalysis( IN start_date DATE, IN end_date DATE ) BEGIN SELECT c.customer_id, CONCAT(c.first_name, ' ', c.last_name) AS customer_name, COUNT(o.order_id) AS orders_in_period, SUM(o.total_amount) AS total_spent FROM customers c JOIN orders o ON c.customer_id = o.customer_id WHERE o.order_date BETWEEN start_date AND end_date GROUP BY c.customer_id, c.first_name, c.last_name ORDER BY total_spent DESC; END // DELIMITER ;
